学习Markdown

Markdown介绍

  • Markdown 是轻量标记语言，2004年被 John Gruber 创造。它允许人们使用 易读易写的纯文本格式编写文档 。
  • Markdown 最重要的设计就是 可读性。
  • Markdown 可以选择性的的转换为 HTML 文档格式。
  • 很多网站目前都使用了 Markdown 或者变种，例如Github等大型社区。

Markdown的优点

  • 纯文本，所以兼容性极强，可以用所有文本编辑器打开。
  • 专注于文字写作而不是排版
  • 格式转换方便，可以轻松转换为 HTML、电子书等。
  • Markdown 标记语法简单好记，可读性强。

Markdown实时预览编辑器

  • 在线编辑器：dillinger
  • 马克飞象
  • MarkdownPad

Markdown 常用语法

标题

  • 以 # 开头，# 表示一级标题， ## 二级标题，依此类推。
  • 书写建议：# 后面加一个空格再写标题内容，标题与标题之间保留换行。

加粗和斜体

  • 加粗
    • 在要加粗的文字两端加上**
  • 倾斜
    • 在要倾斜的文字两端加上*
  • 最好在加粗/倾斜的文字之前加一个空格，看起来更清晰。

无序列表和有序列表

无序列表和有序列表都是按照缩进进行嵌套的。

无序列表

  • 以 - 开头
  • - 之后跟空格 后面是内容
  • 可以添加子列表，添加方式与父列表相同，在父列表下进行缩进即可
  • Markdown 中的无序列表对应的就是 HTML 中的 ul 标签

有序列表

  • 以 1 开头
  • 1 之后跟空格 后面是内容
  • 可以添加子列表，添加方式与父列表相同，在父列表下进行缩进即可
  • Markdown 中的无序列表对应的就是 HTML 中的 ol 标签

段落

  • 在段落中写英文的时候，要注意两边加上空格，为了保持格式清晰
  • 段落直接书写
  • 段落与段落之间需要使用换行

github使用入门

介绍

github 是为开发者提供 GIT 仓库的托管服务。是一个让开发者与朋友、同事及陌生人共享代码的完美平台。

基本使用

Git 与 Github 的区别

  • 在 GIT 中，开发者将原代码存入名叫“Git仓库”的资料库中并加以使用。
  • Github 为开发者的“Git仓库”提供了一个托管的平台，在该平台上公开的软件原代码全都由Git进行管理。
  • Github 除提供Git仓库的托管服务外，还为开发者提供了一系列功能，帮助高效率、高品质的进行代码编写。
  • Github创始人的愿望就是能有一个Git仓库托管服务让自己与朋友轻松分享代码。
